# 📋 Backlog

Le **Backlog** est la vue de gestion de l'ensemble de tes tâches. Contrairement à la [Roadmap](./roadmap) qui est organisée par projet et par étape, le Backlog est une **vue plate filtrée** — toutes tes tâches, triables et filtrables.

:::tip
Utilise la Roadmap pour piloter l'avancement de tes projets, et le Backlog pour retrouver, trier et prioriser des tâches spécifiques.
:::

---

## Créer une tâche rapidement

En haut du Backlog, un **champ de création rapide** te permet d'ajouter une tâche en une ligne :

1. Tape le titre de la tâche
2. Appuie sur **Entrée** ou clique **Ajouter**

La tâche est créée instantanément dans tes actions libres. Tu peux ensuite la rattacher à un projet depuis le Backlog ou depuis la [Roadmap](./roadmap).

---

## Filtres disponibles

Le Backlog propose trois filtres combinables :

| Filtre | Options |
|--------|---------|
| **Projet** | Filtrer par projet spécifique |
| **Catégorie** | Filtrer par catégorie de tâche |
| **Statut** | TODO, En cours, Terminé, etc. |

Le bouton **Réinitialiser les filtres** efface tous les filtres en un clic.

---

## Gérer les tâches depuis le Backlog

Sélectionne une tâche pour accéder aux actions disponibles :
- **Modifier** — éditer titre, priorité, deadline, estimation de temps
- **Compléter** — marquer la tâche comme DONE
- **Déplacer** — rattacher à un projet ou une étape de roadmap
- **Supprimer** — suppression douce (récupérable depuis la Corbeille)

---

## Backlog vs Actions libres

| Backlog | Actions libres |
|---------|----------------|
| Toutes les tâches (planifiées + libres) | Uniquement les tâches sans étape assignée |
| Filtrable par projet, statut, catégorie | Vue spécifique aux tâches non planifiées |
| Vue de gestion générale | Vue "file d'attente" |

Pour les tâches capturées rapidement et pas encore planifiées, voir aussi [Actions libres](./actions-libres).
